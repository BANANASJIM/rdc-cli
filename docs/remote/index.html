Remote & Split Mode

Remote Capture

Use rdc remote to capture from a remote RenderDoc target and pull the .rdc file back locally. 

rdc remote connect 10.0.0.219:39920
rdc remote list
rdc remote capture /usr/bin/vkcube -o /tmp/remote.rdc

Split Mode

Split mode lets a thin client (for example macOS) connect to a daemon that has RenderDoc installed. 

# replay host (Linux/Windows)
rdc capture --output /tmp/game.rdc -- /path/to/game
rdc open /tmp/game_frame0.rdc --listen 0.0.0.0:54321

# thin client (macOS)
PYTHONPATH=src .venv/bin/rdc open --connect host:54321 --token TOKEN
PYTHONPATH=src .venv/bin/rdc capture --output /tmp/mac_capture.rdc -- /usr/bin/vkcube
PYTHONPATH=src .venv/bin/rdc close --shutdown

Notes

  • In split mode, capture and remote commands are routed through daemon RPC.
  • The resulting .rdc file is downloaded back to the client output path.
  • No local RenderDoc installation is required on split clients.
